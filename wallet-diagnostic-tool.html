1 Find Your Completed Bookings
This query shows all recently completed bookings. Find your booking and copy its ID. 
SELECT 
  id,
  vendor_id,
  couple_name,
  service_type,
  status,
  amount,
  fully_completed_at
FROM bookings
WHERE status = 'completed'
ORDER BY fully_completed_at DESC
LIMIT 5;
Expected Result: You should see your recently completed booking(s).
Action: Copy the id value (UUID format) for the next steps.
2 Enter Your Booking ID
Paste the booking ID from Step 1 here. This will be used in the following queries.
üí° The queries below will automatically update when you paste your booking ID
3 Check Wallet Transaction Status CRITICAL CHECK
This query checks if a wallet transaction was created for your booking. 
SELECT 
  b.id as booking_id,
  b.couple_name,
  b.amount as booking_amount,
  b.fully_completed_at,
  wt.transaction_id,
  wt.amount as wallet_amount,
  wt.created_at as transaction_created_at,
  CASE 
    WHEN wt.id IS NULL THEN '‚ùå MISSING'
    ELSE '‚úÖ EXISTS'
  END as wallet_status
FROM bookings b
LEFT JOIN wallet_transactions wt ON wt.booking_id = b.id
WHERE b.id = 'YOUR_BOOKING_ID';
Expected Results:
‚úÖ EXISTS Transaction was created successfully!
‚ùå MISSING Transaction is missing! Continue to Step 4.
4 Check Payment Receipts
If transaction is missing, check if payment receipts exist. 
SELECT 
  id,
  receipt_number,
  payment_type,
  amount,
  payment_method,
  payment_intent_id,
  created_at
FROM receipts
WHERE booking_id = 'YOUR_BOOKING_ID'
ORDER BY created_at;
Expected Result: You should see at least one receipt record.
If Empty: The booking was completed without payment records! This is why the transaction wasn't created.
5 Full Diagnostic Report
This query shows the complete picture: booking, receipts, and wallet transaction status. 
SELECT 
  b.id as booking_id,
  b.vendor_id,
  b.couple_name,
  b.service_type,
  b.status,
  b.amount as booking_amount,
  b.fully_completed_at,
  (SELECT COUNT(*) FROM receipts WHERE booking_id = b.id) as receipts_count,
  (SELECT SUM(amount) FROM receipts WHERE booking_id = b.id) as total_paid_centavos,
  wt.transaction_id,
  wt.amount as wallet_amount,
  wt.created_at as transaction_created_at,
  CASE 
    WHEN wt.id IS NULL THEN '‚ùå MISSING'
    ELSE '‚úÖ EXISTS'
  END as wallet_status
FROM bookings b
LEFT JOIN wallet_transactions wt ON wt.booking_id = b.id
WHERE b.id = 'YOUR_BOOKING_ID';
Interpretation:
‚Ä¢ receipts_count: Number of payment receipts (should be > 0)
‚Ä¢ total_paid_centavos: Total paid amount in centavos (divide by 100 for PHP)
‚Ä¢ wallet_status: Whether transaction exists
‚Ä¢ wallet_amount: Amount transferred to wallet (should match total paid)
6 All Completed Bookings Overview
Quick overview of all completed bookings and their wallet transaction status. 
SELECT 
  b.couple_name,
  b.service_type,
  b.fully_completed_at,
  CASE WHEN wt.id IS NULL THEN '‚ùå MISSING' ELSE '‚úÖ EXISTS' END as wallet_status,
  wt.amount as wallet_amount
FROM bookings b
LEFT JOIN wallet_transactions wt ON wt.booking_id = b.id
WHERE b.status = 'completed'
ORDER BY b.fully_completed_at DESC;
This shows which bookings have wallet transactions and which don't.
