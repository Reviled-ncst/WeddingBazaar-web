The cleanest solution is to modify the backend bookings endpoint to:

1. Extract user ID from the JWT token (already authenticated)
2. Use that user ID automatically (no need to send couple_id)
3. This is more secure and prevents user ID spoofing

Backend code change:
```javascript
// In bookings.cjs POST /api/bookings
const userId = req.user.id;  // From JWT token (already available)
const coupleId = req.body.couple_id || userId;  // Use from body OR from token
```

This way, couple_id becomes OPTIONAL in the request.
