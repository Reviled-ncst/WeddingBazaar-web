🔧 Database Field Mapping Fix - Complete

🚨 Root Cause Found

THE ISSUE: Frontend was sending role: "vendor" but backend expects user_type: "vendor". This field mismatch caused users to not be stored properly in the database.

✅ After (Correct Field Names)

{ "first_name": "John", "last_name": "Doe", "email": "john@example.com", "user_type": "vendor", ✅ Correct! "business_name": "My Business" }
